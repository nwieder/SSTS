Detailed Description

Subunit switch reaction event.

Coordinates reaction events that change the state of a subunit. Whenever a subunit becomes activated the Channl_Spc has to be notified, since the number of open channels depends on the number of active subunits.

void SubUnitSwitch_Rct_Evt::execute ( )
virtual

Implementation of the _Event::execute() method.

Whenever a subunit switch reaction event occurs, a channel subunit switches between an activated state and an inactivated state. This affects the state of a channel and thus needs to be communicated. This happens via a typecast of the _Species pointer to its more specific derivative Channel_Spc (identified via the ch_spc_id attribute). The result of the typecast allows for the call of public functions that are only accessible via the Channel_Spc interface. Consequently the function Channel_Spc::activate_Subunit() or Channel_Spc::inactivate_Subunit() can be called.

Reimplemented from nw::Reaction_Evt.
